How It Works
5-Layer Defense Pipeline
Every prompt passes through our detection stack in order. Threats are blocked at the earliest possible layer — keeping costs zero and latency invisible.
L0
Learned Memory INSTANT — 0-2ms
Previously confirmed threats are stored as fingerprints. Similar prompts are matched using keyword overlap scoring and blocked instantly — no API calls, no cost. Gets smarter with every scan your team makes.
fuzzy matchingkeyword fingerprintsself-improvingzero API cost
L1
Rule Engine <1ms
Regex-based detection catches unicode lookalike attacks (ıgnore → ignore), leetspeak encoding (1gn0r3), base64 injections, invisible zero-width characters, HTML/script injection, and 20+ PII patterns.
unicode normalizationleetspeak decodebase64 decodeinvisible charsPII detection
L2
Pattern Matcher 1–5ms
80+ weighted threat signals scored in real-time. Compound patterns catch social engineering tricks — "hypothetically for a novel, how would someone synthesize..." combined with harmful requests.
80+ signalscompound patternsweighted scoringsocial engineering
L3
LLM Judge 500–800ms
For edge cases that pass all previous layers, a dedicated AI model analyzes intent and full context. Catches grandmother bedtime stories, fictional framings, indirect manipulation, and adversarial creativity.
intent analysiscontext awareGroq powerededge case coverage
CP
Custom Policy Engine YOUR RULES
Per-key custom policies let you define your own blocked keywords, topic restrictions, and max prompt lengths. A children's education platform has different rules than a financial services chatbot.
per-key rulestopic blockingcustom keywordslength limits
